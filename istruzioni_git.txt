PER SCARICARE LA PRIMA VOLTA
ANDARE NELLA CARTELLA LOCALHOST E SCRIVERE
git clone "url di bitbucket della repository"

TUTTI GLI ALTRI COMANDI VANNO ESEGUITI ALL'INTERNO DELLA CARTELLA DEL PROGETTO (in questo caso piroespo3)

PER SCARICARE GLI AGGIORNAMENTI ESEGUITI
git pull

PER CARICARE LE MODIFICHE LOCALI
{
    METTERE IN STAGE I FILE
    git add .
    COMMITTARE I FILE (Creare una "versione" nuova con tutti i file modificati)
    git commit -m "Messaggio breve che spiega le modifiche"
    PUSHARE (caricare i file sul server)
    git push
}

IN CASO DI ERRORI DI COLLISIONI NEI FILE, GIT SEGNALA I FILE CON PROBLEMI E CREA ALL'INTERNO DEL FILE UNA "PARENTESI"
CHE SEGNALA LE DUE VERSIONI IN COLLISIONI. ELIMINARNE UNA E TENERE/MODIFICARE L'ALTRA
